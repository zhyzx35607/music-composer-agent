# 给后端的音乐生成转换模块：最小接入说明

## 1. 先说结论

后端不需要自己写 `json_to_midi`，也不需要自己研究 FluidSynth。

你把整个目录交给后端：

```text
F:\music\gpt_music_pipeline
```

后端只需要调用这个入口：

```text
F:\music\gpt_music_pipeline\run_music_pipeline.py
```

它会自动完成：

```text
用户需求
-> prompt 优化
-> GPT API
-> 保存 music_json
-> music_json 转 MIDI
-> MIDI 转 WAV
-> 输出 manifest
```

## 2. GPT 返回的 JSON 到哪里了

GPT API 返回的 JSON 一开始在程序内存里。

`run_music_pipeline.py` 会立刻把它保存成文件：

```text
F:\music\gpt_music_pipeline\outputs\{version_id}.json
```

例如：

```text
F:\music\gpt_music_pipeline\outputs\gpt_polished_60s.json
```

所以不是“GPT 返回后不知道去哪了”，而是：

```text
GPT 返回内容
-> Python 解析成 dict
-> 校验
-> 写入 outputs/{version_id}.json
```

## 3. JSON 怎么转 MIDI

转换脚本是：

```text
F:\music\gpt_music_pipeline\music_json_to_midi.py
```

单独调用方式：

```powershell
cd F:\music\gpt_music_pipeline

F:\music\.venv\Scripts\python.exe .\music_json_to_midi.py `
  .\outputs\gpt_polished_60s.json `
  .\outputs\gpt_polished_60s.mid
```

输入：

```text
outputs/gpt_polished_60s.json
```

输出：

```text
outputs/gpt_polished_60s.mid
```

## 4. MIDI 怎么转 WAV

转换脚本是：

```text
F:\music\gpt_music_pipeline\render_wav.py
```

它会调用本地 FluidSynth：

```text
F:\music\tools\fluidsynth\fluidsynth-v2.5.6-win10-x64-cpp11\bin\fluidsynth.exe
```

并使用音色库：

```text
F:\music\soundfonts\FluidR3_GM.sf2
```

单独调用方式：

```powershell
cd F:\music\gpt_music_pipeline

F:\music\.venv\Scripts\python.exe .\render_wav.py `
  .\outputs\gpt_polished_60s.mid `
  .\outputs\gpt_polished_60s.wav `
  --trim-seconds 60
```

输入：

```text
outputs/gpt_polished_60s.mid
```

输出：

```text
outputs/gpt_polished_60s.wav
```

## 5. 后端最推荐的接法

后端不要分开调用三个脚本，直接调用总控脚本：

```powershell
cd F:\music\gpt_music_pipeline

F:\music\.venv\Scripts\python.exe .\run_music_pipeline.py `
  --request "生成一段30秒纯音乐，适合作为平台展示背景音乐。" `
  --duration 30 `
  --style "cinematic game background music" `
  --mood "uplifting, polished, gentle energy" `
  --model "gpt-5.5" `
  --timeout 240 `
  --output-name "version_001"
```

这个命令会一次性生成：

```text
outputs/version_001.json
outputs/version_001.mid
outputs/version_001.wav
outputs/version_001.manifest.json
```

## 6. 后端怎么返回给前端

`run_music_pipeline.py` 会打印一段 manifest JSON。

核心字段是：

```json
{
  "success": true,
  "version_id": "version_001",
  "urls": {
    "music_json": "/outputs/version_001.json",
    "midi": "/outputs/version_001.mid",
    "wav": "/outputs/version_001.wav",
    "manifest": "/outputs/version_001.manifest.json"
  }
}
```

后端只要把里面的 `urls.wav` 返回给前端：

```json
{
  "success": true,
  "version_id": "version_001",
  "wav_url": "/outputs/version_001.wav",
  "music_json_url": "/outputs/version_001.json",
  "midi_url": "/outputs/version_001.mid"
}
```

前端拿到后直接播放：

```tsx
<audio controls src={result.wav_url} />
```

## 7. 如果 GPT 调用由你先完成

如果你已经拿到了 GPT 返回的 `music_json`，并保存成：

```text
outputs/my_music.json
```

那就不用再调用 GPT，直接运行：

```powershell
cd F:\music\gpt_music_pipeline

F:\music\.venv\Scripts\python.exe .\run_music_pipeline.py `
  --input-json .\outputs\my_music.json `
  --output-name "my_music"
```

它会自动生成：

```text
outputs/my_music.mid
outputs/my_music.wav
outputs/my_music.manifest.json
```

## 8. 后端必须拿到哪些文件

后端要想真的能跑，至少需要这些：

```text
F:\music\gpt_music_pipeline\run_music_pipeline.py
F:\music\gpt_music_pipeline\gpt_music_client.py
F:\music\gpt_music_pipeline\prompt_builder.py
F:\music\gpt_music_pipeline\validate_music_json.py
F:\music\gpt_music_pipeline\music_json_to_midi.py
F:\music\gpt_music_pipeline\render_wav.py
F:\music\gpt_music_pipeline\prompts\
F:\music\gpt_music_pipeline\schemas\
F:\music\tools\fluidsynth\
F:\music\soundfonts\FluidR3_GM.sf2
```

如果后端没有这些文件，它当然转不了 MIDI/WAV。

## 9. 你和后端的分工

你负责交付：

```text
音乐生成转换模块
prompt 优化
GPT 调用
music_json 校验
json_to_midi
midi_to_wav
manifest 输出
```

后端负责：

```text
提供 /api/generate
接收前端参数
调用 run_music_pipeline.py
读取 manifest
把 /outputs/*.wav 返回给前端
静态托管 outputs 目录
```

前端负责：

```text
输入需求
调用 /api/generate
展示生成中
播放 wav_url
下载 wav
```

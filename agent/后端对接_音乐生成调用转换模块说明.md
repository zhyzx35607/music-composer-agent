# 后端对接：音乐生成调用转换模块说明

## 1. 目标

前端提交音乐需求后，后端调用本模块完成：

```text
前端需求
-> prompt 优化
-> GPT API
-> music_json
-> json_to_midi
-> FluidSynth
-> wav
-> 返回前端播放地址
```

本模块位置：

```text
F:\music\gpt_music_pipeline
```

统一输出目录：

```text
F:\music\gpt_music_pipeline\outputs
```

后端只需要把该目录映射成静态访问路径：

```text
/outputs
```

## 2. 职责划分

### 2.1 调用转换模块负责

由音乐生成功能负责人维护，也就是当前这部分：

- prompt 优化模板。
- 调用 GPT API 生成 `music_json`。
- 校验 `music_json`。
- 把 `music_json` 转成 MIDI。
- 调用 FluidSynth 把 MIDI 转成 WAV。
- 生成 manifest，告诉后端 JSON/MIDI/WAV 文件路径和前端访问 URL。

核心入口脚本：

```text
F:\music\gpt_music_pipeline\run_music_pipeline.py
```

### 2.2 后端负责

后端同学需要做：

- 提供 `POST /api/generate` 接口给前端调用。
- 接收前端参数，例如需求文本、时长、风格、情绪。
- 在服务器环境变量中配置 API key 和中转地址。
- 调用 `run_music_pipeline.py`。
- 读取脚本输出的 manifest JSON。
- 把 `wav_url`、`json_url`、`midi_url` 等结果返回给前端。
- 把 `F:\music\gpt_music_pipeline\outputs` 作为静态文件目录暴露为 `/outputs`。

后端不需要自己理解 MIDI 格式，也不需要自己调用 FluidSynth。

## 3. 环境变量

后端启动时必须配置：

```powershell
$env:MUSIC_API_KEY = "实际 key"
$env:MUSIC_API_BASE_URL = "http://172.93.218.157:8080/v1"
$env:MUSIC_MODEL = "gpt-5.5"
```

注意：

- key 不允许写死在前端。
- key 不允许写死在代码仓库。
- 可以放在 `.env`、系统环境变量或后端配置中心。

## 4. 后端调用命令

示例：生成 30 秒音乐。

```powershell
cd F:\music\gpt_music_pipeline

F:\music\.venv\Scripts\python.exe .\run_music_pipeline.py `
  --request "生成一段30秒纯音乐，适合作为平台展示背景音乐。" `
  --duration 30 `
  --style "cinematic game background music" `
  --mood "uplifting, polished, gentle energy" `
  --model "gpt-5.5" `
  --timeout 240 `
  --output-name "version_001"
```

示例：只把已有 `music_json` 转成 MIDI/WAV。

```powershell
cd F:\music\gpt_music_pipeline

F:\music\.venv\Scripts\python.exe .\run_music_pipeline.py `
  --input-json .\outputs\gpt_polished_60s.json `
  --output-name "version_002"
```

  "duration_seconds": 60,
  "tempo_bpm": 120,

然后返回给前端。

## 6. 推荐后端接口

### 6.1 生成音乐

```http
POST /api/generate
Content-Type: application/json
```

请求体：

```json
{
  "prompt": "生成一段适合游戏主菜单的背景音乐",
  "duration_seconds": 30,
  "style": "cinematic game background music",
  "mood": "uplifting, polished, gentle energy"
}
```

成功响应：

```json
{
  "success": true,
  "version_id": "version_001",
  "title": "Hopeful Circuit",
  "duration_seconds": 30,
  "music_json_url": "/outputs/version_001.json",
  "midi_url": "/outputs/version_001.mid",
  "wav_url": "/outputs/version_001.wav",
  "manifest_url": "/outputs/version_001.manifest.json"
}
```

失败响应：

```json
{
  "success": false,
  "error_code": "GENERATION_FAILED",
  "message": "GPT API request timed out"
}
```

### 6.2 查询版本

```http
GET /api/version/{version_id}
```

响应：

```json
{
  "success": true,
  "version_id": "version_001",
  "music_json_url": "/outputs/version_001.json",
  "midi_url": "/outputs/version_001.mid",
  "wav_url": "/outputs/version_001.wav",
  "manifest_url": "/outputs/version_001.manifest.json"
}
```

## 7. 前端如何播放

前端调用：

```text
POST /api/generate
```

拿到：

```json
{
  "wav_url": "/outputs/version_001.wav"
}
```

然后直接：

```html
<audio controls src="/outputs/version_001.wav"></audio>
```

或者 React 中：

```tsx
<audio controls src={result.wav_url} />
```

## 8. 演示链路

演示时可以这样讲：

```text
1. 用户在前端输入需求。
2. 后端收到需求，调用音乐生成模块。
3. 音乐生成模块先把需求整理成更适合 GPT 的 prompt。
4. GPT 返回严格格式的 music_json。
5. 本地校验器检查 music_json 是否能转 MIDI。
6. json_to_midi 把 music_json 转成 .mid。
7. FluidSynth 使用 SoundFont 把 .mid 渲染成 .wav。
8. 后端把 wav_url 返回给前端。
9. 前端播放 /outputs/version_001.wav。
```

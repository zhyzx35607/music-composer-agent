# AI音乐生成系统流程与分工文档

## 一、项目目标

本项目拟实现一个基于 AI 的纯音乐生成系统。用户在前端输入自然语言音乐需求后，系统会对需求进行整理和增强，再调用 GPT 生成结构化音乐数据，随后由后端将结构化音乐数据转换为 MIDI 文件，并使用 FluidSynth 将 MIDI 渲染为 WAV 音频，最终返回前端供用户播放或下载。

本系统的核心 AI 参与点在于：GPT 根据用户自然语言需求生成音乐结构，包括节奏、速度、调性、和弦、旋律、乐器轨道和音符信息。

## 二、整体流程说明

### 1. 用户输入阶段

用户在前端页面输入音乐需求，例如：

> 生成一段 30 秒左右的纯音乐，风格紧张一点，适合游戏 Boss 战。

前端负责收集用户输入，并可提供基础参数选择，例如音乐时长、风格、情绪、乐器偏好等。

### 2. 需求整理与提示词优化阶段

后端接收到用户需求后，对需求进行规范化处理，包括：

- 判断用户是否指定时长、风格、情绪、乐器、速度、调性等信息。
- 对缺失信息进行默认补全，例如默认时长 30 秒、默认 4/4 拍、默认纯音乐。
- 将用户自然语言需求转化为更适合 GPT 作曲的提示词。

示例：

```text
请根据用户需求生成一段结构化纯音乐数据。
要求输出 JSON，包含 tempo_bpm、key、time_signature、tracks、notes 等字段。
音乐时长约 30 秒，不包含人声，适合游戏 Boss 战场景。
```

### 3. GPT 生成音乐结构阶段

系统通过 API 调用 GPT。GPT 不直接生成 WAV 音频，而是生成结构化音乐 JSON，例如：

```json
{
  "title": "Boss Battle Loop",
  "duration_seconds": 30,
  "tempo_bpm": 128,
  "key": "D minor",
  "time_signature": "4/4",
  "tracks": [
    {
      "name": "Melody",
      "instrument": "lead synth",
      "notes": []
    },
    {
      "name": "Bass",
      "instrument": "electric bass",
      "notes": []
    }
  ]
}
```

这种方式比让 GPT 直接输出二进制 MIDI 文件更稳定，也方便后端校验和转换。

### 4. JSON 转 MIDI 阶段

后端读取 GPT 返回的 music_json，并将其中的音符、轨道、速度、乐器等信息转换为标准 MIDI 文件。

转换内容包括：

- tempo_bpm 转换为 MIDI tempo meta event。
- time_signature 转换为 MIDI 拍号信息。
- tracks 转换为多个 MIDI 轨道。
- notes 转换为 note_on 和 note_off 事件。
- instrument 转换为 MIDI program change。

输出结果为：

```text
output.mid
```

### 5. MIDI 转 WAV 阶段

后端使用 FluidSynth 和 SoundFont 将 MIDI 渲染为 WAV 音频。

示例命令：

```powershell
fluidsynth -ni -F output.wav -T wav -O s16 -r 44100 soundfont.sf2 output.mid
```

输出结果为：

```text
output.wav
```

### 6. 前端展示阶段

后端将 WAV 文件地址返回给前端。前端提供音频播放器，使用户可以试听生成结果，也可以下载 WAV 或 MIDI 文件。

完整链路如下：

```text
用户输入需求
-> 前端提交
-> 后端整理需求
-> 生成 GPT 提示词
-> GPT 返回 music_json
-> 后端转换 MIDI
-> FluidSynth 渲染 WAV
-> 前端播放和下载
```

## 三、三人分工表与中期检查效果

|---|---|---|---|

| 成员三 | 智能体开发 | 设计 GPT 提示词模板，完成用户需求到音乐生成提示词的转换，定义 music_json 输出格式，并对 GPT 返回内容进行校验。 | 能够根据不同用户需求生成结构化 music_json，包含时长、速度、调性、乐器、轨道和音符信息。 |

| 成员   | 原分工    | 调整后分工                  | 主要任务                                                     |
| ------ | --------- | --------------------------- | ------------------------------------------------------------ |
| 成员一 | 前端      | 前端与联调                  | 保持原前端页面不变，负责调用新的 `/api/generate` 接口，展示生成状态，播放 WAV，提供下载按钮。 |
| 成员二 | 后端      | 后端接口与流程编排          | 负责接收前端请求、调用智能体模块、保存 JSON/MIDI/WAV 文件、调用转换模块、把结果地址返回给前端。 |
| 成员三 | Text2midi | AI 音乐智能体与音乐生成模块 | 放弃 Text2midi，改为负责 GPT 提示词设计、music_json 格式设计、GPT 返回内容校验、JSON 转 MIDI、MIDI 转 WAV 的音乐生成链路。 |

### 中期检查目标

中期阶段需要完成一个最小可运行闭环：

```text
固定测试需求
-> GPT 或测试 JSON
-> MIDI 文件
-> WAV 文件
```

中期检查重点：

- 前端页面可以提交需求。
- 后端可以接收请求。
- 智能体可以输出结构化 music_json。
- 系统可以根据 music_json 生成 MIDI。
- 系统可以将 MIDI 转为 WAV。
- 至少生成一段约 30 秒的纯音乐音频。

### 最终检查目标

最终阶段需要完成完整系统：

```text
用户输入自然语言需求
-> 系统自动优化提示词
```

最终检查重点：

- 用户输入不同类型需求时，系统都能生成对应音乐。

- 系统能够说明 AI 参与方式，而不是简单播放固定音频。

## 六、项目亮点

本项目不是简单调用现成音乐文件，而是通过 GPT 将自然语言需求转化为结构化音乐创作数据，再由程序自动转换为可播放音频。系统中 AI 参与了音乐创作的核心环节，包括风格理解、参数补全、旋律生成、和弦设计和多轨编排，具有较清晰的智能体参与过程和完整工程链路。
